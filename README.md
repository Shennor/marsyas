Данный проект представляет собой систему автоматического анализа логов тестирования дистрибутива на основе экспертной системы. 

Основные функции имеют два типа:
Filler (kbs\_text, log\_info) -> new\_kbs\_text - программа, наполняющая базу знаний объектами на основе присутствующих логов тестирования.
Parser (solver, log_info) -> solver - программа, по логам заполняющая рабочую память системы.
Система поддерживает добавление, включение и отключение модулей с различными filler и parser. 
Новые модули с именем X.py должны содержать списки с именами X\_parsers и X\_fillers, которые содержат соответственно имена всех парсеров и филлеров, реализованных в модуле.
Для включения/выключения модулей необходимо в файле modues.txt добавить/удалить имя модуля.
